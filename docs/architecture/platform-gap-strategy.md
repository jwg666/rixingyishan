# 平台差异与降级策略（App / H5 / 小程序）

## 能力差异矩阵
| 能力 | iOS/Android App | H5 | 微信小程序 | 统一策略 |
|---|---|---|---|---|
| 拍照 | 原生支持，稳定 | 依赖浏览器能力 | 支持 | 优先统一 `takePhoto()`，失败时引导相册选择 |
| 录视频 | 原生支持，参数可控 | 浏览器兼容性不一 | 支持但参数受限 | 统一 `recordVideo()`，对时长/清晰度做平台兜底 |
| 文件系统 | 能力强 | 受限 | 可用但路径规则不同 | 统一文件抽象层，不在业务代码存平台路径 |
| 后台上传 | 可一定程度支持 | 浏览器标签页受限 | 生命周期受限 | 任务持久化 + 前台恢复上传，不承诺强后台 |
| 权限弹窗 | OS 级别 | 浏览器弹窗与策略差异 | 平台授权模型 | 权限预检测 + 用户可理解提示 |

## 差异处理原则
- `能力探测优先`：进入新建页时探测关键能力（相机、录像、文件、网络）
- `优雅降级`：录像不可用时降级为拍照+文字；拍照不可用时仅文字
- `统一报错语义`：将平台错误码映射成统一错误类型（权限拒绝、能力缺失、文件异常、网络异常）

## 统一接口建议
```ts
interface PlatformMediaAdapter {
  canTakePhoto(): Promise<boolean>;
  canRecordVideo(): Promise<boolean>;
  takePhoto(): Promise<{ localPath: string; width?: number; height?: number }>;
  recordVideo(): Promise<{ localPath: string; durationMs?: number; sizeBytes?: number }>;
}
```

## 用户体验策略
- 首次失败给出“原因 + 解决建议”，避免只弹技术错误
- 同一权限失败不要重复轰炸弹窗，保留“去设置开启权限”入口
- 对录视频默认限制时长，降低上传失败率

## 质量基线（MVP）
- 同一功能在 App/H5/小程序均有可达路径
- 任一端能力缺失不阻断“文字记录”主流程
- 同类错误展示文案一致，便于客服与运营定位问题
